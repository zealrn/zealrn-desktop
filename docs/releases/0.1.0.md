# ZealRN 0.1.0

Release date: 2026-07-15

## Highlights

- Separate ZealRN application identity and local data paths.
- Persistent Start Note available before any documentation is opened.
- Page-linked SQLite learning notes with autosave and selected-text capture.
- All Notes browser with search, edit, delete, and source-page reopening.
- Markdown, searchable PDF, JSON, ZIP, and raw database backup exports.
- Offline CodeMirror web playground with isolated preview and console.
- System, Light, and Dark appearance.
- Embedded Developer Terminal with bundled xterm.js and a native Linux PTY; Windows uses external-terminal integration.
- Expanded Learning Notes editor with Markdown tools, preview, search, zoom, and focus modes.
- First-run Quick Tour, welcome checklist, contextual tips, and offline Help pages.

## Installation

Recommended user-local installation without sudo:

```bash
./scripts/install-user-local.sh
~/.local/bin/zealrn
```

Portable launch:

```bash
chmod +x dist/ZealRN-0.1.0-x86_64.AppImage
./dist/ZealRN-0.1.0-x86_64.AppImage
```

Optional system installation on Zorin/Ubuntu:

```bash
sudo apt install ./dist/zealrn_0.1.0_amd64.deb
```

## Uninstallation

User-local install:

```bash
./scripts/uninstall-user-local.sh
```

Debian package:

```bash
sudo apt remove zealrn
```

Uninstallation intentionally leaves user settings, notes, exports, and docsets in place.

## Security Notes

Web Playground code runs locally in an isolated preview that blocks external network requests, popups, permissions, and external navigation. It is not a perfect security sandbox.

The Developer Terminal is not sandboxed and has the same access as the user's normal shell. The terminal shows a one-time warning before use.

## Known Limitations

- One main plain-text note per documentation page.
- No sync, accounts, cloud storage, note import, highlights, bookmarks, or flashcards.
- AppImage requires normal Linux desktop baseline libraries; it is not intended for a bare server image.
- The Developer Terminal has normal user-shell access and is not sandboxed; the external-terminal action remains available.
- Downloaded documentation theme changes may require restart with Qt 6.4-6.6.
- Windows packages are unsigned builds and may trigger SmartScreen warnings.
