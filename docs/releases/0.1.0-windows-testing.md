# ZealRN 0.1.0 Windows Testing

This release targets Windows 10/11 x64. It opens an external Windows Terminal, PowerShell, Command Prompt, or Git Bash session instead of embedding ConPTY. The installer is unsigned, so Microsoft Defender SmartScreen may show a warning. Do not disable Windows security; verify the checksum before running the file.

## Files

- `ZealRN-0.1.0-win64-setup.exe`: current-user installer, no administrator access required for the default location.
- `ZealRN-0.1.0-win64-portable.zip`: portable build that stores its configuration beside the executable.
- `SHA256SUMS`: SHA-256 hashes for both artifacts.

## Verify

Open PowerShell in the folder containing the copied files:

```powershell
Get-FileHash .\ZealRN-0.1.0-win64-setup.exe -Algorithm SHA256
Get-FileHash .\ZealRN-0.1.0-win64-portable.zip -Algorithm SHA256
Get-Content .\SHA256SUMS
```

The displayed hashes must match `SHA256SUMS` exactly.

## Installer Test

1. Run `ZealRN-0.1.0-win64-setup.exe` and keep the default current-user location.
2. Launch **ZealRN** from the Start Menu.
3. Open **File → Docset Library → Available**, refresh, and download a small docset.
4. Open documentation, close ZealRN, disconnect the network, restart, and confirm the installed docs still open.
5. Create and save a Learning Note, restart, and confirm it remains.
6. Open **All Notes**, search by page title and note content, and reopen the source page.
7. Export the note as Markdown, searchable PDF, and JSON; export all notes as ZIP.
8. Open Web Playground, run the starter HTML/CSS/JavaScript, and test **Open in Browser**.
9. Open Developer Terminal and verify **Open External Terminal** detects `wt.exe`, `pwsh.exe`, `powershell.exe`, `cmd.exe`, or Git Bash when installed.
10. Switch System, Light, and Dark appearance modes and restart after each.
11. Uninstall ZealRN from Windows Settings.
12. Confirm notes/docsets remain and any separately installed original Zeal still works.

## Portable Test

1. Extract `ZealRN-0.1.0-win64-portable.zip` to a writable folder.
2. Run `zealrn.exe`.
3. Repeat the docset, notes, export, playground, appearance, and terminal tests above.
4. Move the extracted folder and confirm it still launches.

## Data Locations

The installed build uses Qt's per-user Windows locations under the current user's profile. Expected roots are:

- Roaming settings/data: `%APPDATA%\abnzrdev\ZealRN\`
- Local cache/data: `%LOCALAPPDATA%\abnzrdev\ZealRN\`

The exact docset directory remains configurable in Preferences. The installer and uninstaller manage only program files and Start Menu integration; they do not remove notes, settings, exports, or docsets.

The portable ZIP intentionally keeps its configuration/data beside `zealrn.exe`. Do not place the portable build under a read-only directory.
